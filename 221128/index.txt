Redux란?
-정의 : 상태 관리 라이브러리
- 상태(state)를 관리해주는 라이브러리. 전역 상태값이라고 생각하면 된다.
-특징? 왜나왔나?
 -Flux 모델을 사용한다.
 -Flux 모델이란 기존에 많이 사용했던 MVC 모델의 단점을 보완하기 위해 개발되었다.
 -MVC 모델 : model view controller 
  -controller => Model <-> view
 -Action -> Dispatch -> Reducer -> Store -> View -> Action
 -store : 저장소 , state들을 모아둔 객체
 -action :요청 ,  요청내용에 대해 목적과 필요한 내용(payload)을 담은 객체
 -dispatch : 요청을 받아 reducer에게 전달하는 함수
 -reducer : 요청을 실행하는 함수

 -Redux의 3가지 원칙

 -Single Source Of Truth(진실은 하나의 근원으로부터) : state를 담고있는 store(저장소)는 하나만 잇어야 한다.
 - State is read-only(상태는 읽기 전용이다.)
 변경(재정의)하는 방법은 action을 dispatch로 전달하는것 뿐이다.

 -Changes are made with pure functions(변화는 순수함수로 작성되어야한다.) : reducer는 순수 함수로 작성되어야한다.
   -순수 함수란 같은 매개 변수가 전달되었을때 항상 같은 값을 반환하는 함수이다.
 - 함수 외부의 변수를 변경하지 않고 외부의 변수에 의존하지 않고 매개변수 또한 변경하지 않는다.
